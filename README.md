# Sirius.Achievements

**Веб-платформа учёта и управления достижениями студентов**

> Курсовой проект — Информационные системы и технологии
> Образовательный центр «Сириус», 2026

---

## Описание

**Sirius.Achievements** — полнофункциональная веб-платформа для учёта, модерации и анализа достижений студентов образовательного центра. Система автоматизирует процесс подачи, проверки и утверждения достижений, формирует рейтинги студентов и генерирует AI-сводки профилей на основе загруженных документов.

### Цель работы

Разработка информационной системы, обеспечивающей цифровизацию процесса учёта студенческих достижений с поддержкой модерации, геймификации и интеллектуального анализа документов.

### Задачи

1. Проектирование архитектуры и базы данных системы
2. Реализация системы аутентификации и авторизации с разграничением ролей
3. Разработка модуля загрузки и модерации достижений
4. Создание рейтинговой системы с сезонной механикой
5. Интеграция OCR-распознавания текста из документов (грамоты, сертификаты)
6. Реализация AI-генерации сводки профиля (YandexGPT)
7. Разработка системы поддержки с чатом в реальном времени
8. Внедрение системы уведомлений через WebSocket
9. Развёртывание в контейнерной среде Docker

## Функциональные возможности

### Роли пользователей

| Роль | Описание | Возможности |
|------|---------|------------|
| **Гость** (GUEST) | Незарегистрированный / ожидающий подтверждения | Регистрация, подтверждение email |
| **Студент** (STUDENT) | Активный участник | Загрузка достижений, просмотр рейтинга, обращения в поддержку, профиль |
| **Модератор** (MODERATOR) | Проверяющий | Модерация достижений и пользователей, чат поддержки, назначение работы |
| **Супер-администратор** (SUPER_ADMIN) | Полный доступ | Управление всей системой, завершение сезонов, CMS-страницы |

### Модуль достижений

- Загрузка документов (PDF, JPEG, PNG, WebP) с валидацией формата и размера
- Категоризация: Спорт, Наука, Искусство, Волонтёрство, Другое
- Уровни: Школьный, Муниципальный, Региональный, Федеральный, Международный
- Автоматический расчёт баллов по уровню достижения
- Статусы: Ожидание → Одобрено / Отклонено / На доработке
- Предпросмотр документов в модальном окне

### Система модерации

- Трёхступенчатый интерфейс: **Входящие** → **Моя работа** → **Все записи**
- Назначение модератора на конкретное достижение/пользователя (кнопка «Взять в работу»)
- Возможность отказаться от взятой работы
- Зональный доступ: модератор видит только свой уровень образования
- Одобрение / отклонение с указанием причины
- Статусы в таблицах: Новый, В работе, Занят

### Рейтинговая система

- Глобальный и лиговый рейтинг студентов
- Фильтрация по уровню образования, курсу и направлению
- Топ-3 с карточками, остальные — в таблице
- Кнопка «Найти меня» — автопрокрутка к позиции пользователя с подсветкой
- Сезонная механика: завершение сезона с фиксацией результатов и обнулением баллов
- Экспорт рейтинга в CSV

### Система поддержки

- Создание тикетов с темой и сообщением
- Чат между студентом и модератором с вложениями файлов
- Назначение модераторов на тикеты
- Автоархивация неактивных тикетов (90 дней)
- Настраиваемые сроки сессий (день, неделя, месяц)

### AI-сводка профиля

- OCR-распознавание текста из загруженных грамот и сертификатов (EasyOCR)
- Извлечение текста из PDF-документов (PyMuPDF)
- Генерация структурированной сводки через YandexGPT API
- Локальная генерация при недоступности API (статистика + список достижений)
- Кэширование результата, регенерация при появлении новых документов

### Уведомления

- Доставка в реальном времени через WebSocket
- Хранение в базе данных с отметками о прочтении
- Ссылки для перехода к событию

### Тёмная тема

- Полная поддержка светлой и тёмной палитры
- Анимированный переключатель (солнце/луна)
- Конфигурируемые цвета через `theme.conf`
- Сохранение выбора в localStorage

## Установка и запуск

### Системные требования

- **Docker** и **Docker Compose** (рекомендуемый способ)
- **4 ГБ RAM** минимум (PyTorch + EasyOCR)
- Модели OCR: `craft_mlt_25k.pth`, `cyrillic_g2.pth` в папке `models/`

### 1. Клонирование репозитория

```bash
git clone https://github.com/RochDM231718/sirius_achievements.git
cd sirius_achievements
```

### 2. Настройка переменных окружения

```bash
cp .env.example .env
```

Откройте `.env` и заполните обязательные параметры:

| Переменная | Описание | Пример |
|-----------|---------|--------|
| `DB_USERNAME` | Пользователь PostgreSQL | `sirius_user` |
| `DB_PASSWORD` | Пароль PostgreSQL | `secure_password` |
| `DB_NAME` | Имя базы данных | `sirius_db` |
| `REDIS_PASSWORD` | Пароль Redis | `redis_password` |
| `SECRET_KEY` | Секретный ключ приложения | Сгенерировать: `openssl rand -base64 32` |
| `ALLOWED_HOSTS` | Разрешённые хосты | `localhost,127.0.0.1` |
| `MAIL_HOST` | SMTP-сервер | `smtp.yandex.ru` |
| `MAIL_USERNAME` | Email для отправки | `support@example.ru` |
| `MAIL_PASSWORD` | Пароль приложения | — |

Опциональные (AI-сводка):

| Переменная | Описание |
|-----------|---------|
| `YANDEX_API_KEY` | API-ключ YandexGPT |
| `YANDEX_FOLDER_ID` | ID каталога в Yandex Cloud |
| `RESUME_EXTERNAL_AI_ENABLED` | Включить YandexGPT (`True`/`False`) |

### 3. Запуск через Docker

```bash
docker compose up -d --build
```

Будут запущены 3 контейнера:

| Контейнер | Сервис | Порт |
|-----------|--------|------|
| `sirius_db_new` | PostgreSQL 15 | 5433 (внутренний) |
| `sirius_redis_new` | Redis 7 | — (внутренний) |
| `sirius_app_new` | FastAPI (Uvicorn) | **8081** |

### 4. Проверка

```bash
# Логи приложения
docker logs sirius_app_new --tail 20

# Ожидаемый вывод:
# Application startup complete.
```

Приложение доступно: **http://localhost:8081/sirius.achievements/**

### 5. Начальные данные (опционально)

```bash
# Применение миграций
docker exec sirius_app_new alembic upgrade head

# Заполнение тестовыми данными
docker exec sirius_app_new python app/seeders/main.py
```


## Конфигурация

### Система начисления баллов

| Уровень достижения | Баллы | Переменная окружения |
|--------------------|---------|--------------------|
| Школьный | 10 | `POINTS_SCHOOL` |
| Муниципальный | 20 | `POINTS_MUNICIPAL` |
| Региональный | 40 | `POINTS_REGIONAL` |
| Федеральный | 75 | `POINTS_FEDERAL` |
| Международный | 100 | `POINTS_INTERNATIONAL` |

### Rate Limiting (защита от перебора)

| Параметр | Значение | Описание |
|---------|---------|---------|
| `LOGIN_MAX_ATTEMPTS` | 5 | Попыток входа до блокировки |
| `LOGIN_LOCKOUT_TTL` | 900 | Блокировка на 15 минут (сек) |
| `UPLOAD_MAX_PER_HOUR` | 20 | Лимит загрузок файлов в час |
| `FORGOT_PWD_MAX_ATTEMPTS` | 5 | Попыток сброса пароля |

### Лимиты загрузки файлов

| Тип файла | Лимит | Переменная |
|----------|-------|-----------|
| Аватар | 2 МБ | `MAX_AVATAR_SIZE` |
| Документ | 10 МБ | `MAX_DOC_SIZE` |
| Файл поддержки | 5 МБ | `MAX_SUPPORT_FILE_SIZE` |
